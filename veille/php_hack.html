Le code source de PHP a été hacké

Rédigé par Thomas Piard le 02 Avril 2021

  • Voir l'article


Lors du weekend du dimanche 28 mars 2021, Nikita Popov, développeur PHP a annoncé que les serveurs Git
du projet avaient été attaqués. Actuellement, aucune conséquences n'a été trouvé mais les serveurs ont
été déplacer sur Github par mesure de sécurité. Apparement, cela viendrais d'une corruption du serveur git.php.net

Ce hack aurais été catastrophique pour des millions de sites web à travers le monde, en effet, les pirates
ont pu insérer dans les serveurs 2 malwares ainsi qu'une backdoor.Les malwares étaient ensuite téléchargeable
au sein du répertoire officiel et prétendais corriger une faute de frappe dans le code source. De plus, les malwares
ont été posté sous le nom de Nikita Popov (cité précedemment) et de son collègue Rasmus Lerdorf.
Fort heureusement, les pirates ainsi que les fichiers ont été repérer rapidement et les fichiers téléchargeable
ont été supprimés. Puis, les créateurs du projet ont déplacés leurs serveurs sur Github
afin de protéger les données.

Ce n'est pas la première fois que cela arrive, en 2019, des pirates avaient remplacer le fichier PEAR (PHP Extension
and Application Repository) par une copie malveillante. Toutefois, Nikita Popov rassure les utilisateurs
en leurs expliquant que tout les répertoires PHP seront analysés pour rechercher d'autres codes malveillants.
Ce qui serais malencontreux car l'utilisation de ce langage de programmation est estimé
à plus de 80% des sites web actuellement en ligne.
